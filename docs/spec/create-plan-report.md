# Create a `{type}` for project `{projectId}` and write it to `{path}`.

## Purpose

Generate and persist a structured markdown document — a plan, a report, or a
timeline — derived from project knowledge and filesystem evidence. These
documents are the primary deliverable of a deep project inspection session.

`{type}` is one of: `plan`, `report`, `timeline`.

---

## Inputs

| Parameter   | Type    | Required | Description                                                                |
| ----------- | ------- | -------- | -------------------------------------------------------------------------- |
| `type`      | string  | Yes      | Document type: `plan`, `report`, or `timeline`.                            |
| `projectId` | long    | Yes      | Registered project ID. Used to retrieve persisted knowledge and summaries. |
| `path`      | string  | Yes      | Destination file path for the generated document.                          |
| `title`     | string  | No       | Document title. Defaults to `"<Type>: <project name>"`.                    |
| `overwrite` | boolean | No       | Overwrite `path` if it already exists. Default: `false`.                   |

---

## Document Types

### `plan`

A structured action plan derived from open work streams identified during
project inspection. Sections:

1. **Objective** — one-paragraph statement of what the plan is for.
2. **Open Work Streams** — bullet list of work items inferred from TODO items,
   open summaries, and known-issues notes.
3. **Prioritised Next Steps** — numbered list, most critical first.
4. **Risks and Blockers** — items that could impede progress.

### `report`

A project status report grounded in persisted folder summaries and project
knowledge. Sections:

1. **Executive Summary** — two to three sentences.
2. **Project Structure** — description of key directories and their purposes.
3. **Current State** — what has been built, tested, and documented.
4. **Gaps** — areas with no summary, minimal coverage, or outstanding issues.
5. **Recommendations** — concrete suggestions for next steps.

### `timeline`

A chronological narrative of project activity derived from git log metadata
and summary timestamps. Sections:

1. **Milestones** — significant commits or summary events, oldest first.
2. **Activity Heatmap** (text form) — which directories saw the most recent
   changes.
3. **Inferred Trajectory** — brief narrative of where the project appears to be
   headed based on recent activity.

---

## Expected Output

The generated markdown is written to `path`. The tool returns:

```
Created <type>: /abs/path/to/document.md  (1 204 chars)
```

---

## Behaviour Rules

1. `projectId` must reference an existing `ProjectFolder`.
2. The document is generated by combining persisted knowledge notes, stored
   folder summaries, and any git metadata available for the project root.
3. Generation uses the configured chat model. If the model is unavailable, the
   tool returns an error rather than writing an incomplete document.
4. When `overwrite` is `false` and `path` exists, return:
   `"Error creating <type>: destination already exists: <absolute-path>"`.
5. The generated markdown must begin with a level-1 heading matching `title`.
6. Parent directories of `path` are created automatically if absent.

---

## Edge Cases

| Situation                                          | Expected behaviour                                                                                       |
| -------------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| No persisted knowledge for `projectId`             | Generate the document from filesystem evidence only; note the absence of cached knowledge in the output. |
| Model returns an empty response                    | Return an error; do not write a blank file.                                                              |
| `path` points inside the project's own source tree | Permitted; the agent is allowed to write reports into the project.                                       |
